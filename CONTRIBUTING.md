# 贡献指南

感谢你有兴趣参与 JellyMate！

## 提 Issue

- **Bug：** 使用 Bug Report 模板，附上 macOS 版本、芯片类型、复现步骤
- **新功能：** 使用 Feature Request 模板，说明使用场景而非只描述功能本身

## 提交 PR

1. Fork 仓库，基于 `main` 新建分支
2. 改动尽量小而聚焦，一个 PR 做一件事
3. 改了交互行为请在 PR 描述里说明前后差异
4. `npm test` 通过后再提交

## 代码风格

- 渲染层（`src/index.html`）：保持内联，避免引入外部依赖
- 主进程（`src/main.js`）：保持轻量，不做业务逻辑
- 不写解释"做什么"的注释，只写解释"为什么"的注释

## 关于架构

这是一个刻意选择"无框架"的项目。如果你想重构为 React/Vue/Vite，请先开 Issue 讨论——这不是技术懒惰，是有意为之的设计取舍。
