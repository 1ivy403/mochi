# 情绪小兽 · 串联界面(空壳版)设计文档

日期:2026-06-08
状态:已与产品对齐,待实现

## 一句话目标

把现在「活的桌面水母」和「已做好的面板原型 diary-tab-v5」接通,做出一个能在桌面上**点开水母 → 看到聊天+日记面板 → 再点收起**的完整空壳。数据先用假的,聊天先不接真 AI。

## 三块东西的关系(背景)

| 名字 | 是什么 | 处置 |
|------|--------|------|
| mochi(桌面宠物) | `src/index.html`,活的浮动水母 | 保留,清理行为逻辑 |
| interactions-v2.html | 交互剧本(冒泡/生成卡/明信片/面板朝向) | 作为行为参考 |
| diary-tab-v5.html | 320×640 真面板 UI(聊天+日记,假数据) | 嵌进桌面层,作为面板 |

## 已拍板的决策

1. 面板形态 = diary-tab-v5 的 320×640 竖板,行为遵循 interactions-v2。
2. **架构:面板画在水母同一张全屏透明层里(浮层),不开新窗口。**
3. ✦「生成今日情绪卡片」入口放在聊天输入框右侧。
4. **宠物身体动画 = 3 个「在场态」:安静 / 开心 / 想你,按「上次互动时间」切换**(遵循 triggers.md / 主文档 5.1)。
   - `开心` = 近期有互动;`安静` = 超过 2 小时无互动;`想你` = 超过 1 天无互动。
   - **情绪(平静/开心/焦虑/低落)只体现在卡片和走势图的颜色上,不画在宠物身体上**——这两套是不同的轴,之前被混成一套了,现已纠正。
   - 素材映射(已定):`安静=mochi_body(calm)`、`开心=mochi_happy`、`想你=mochi_low(雾紫,紫色)`。`mochi_anxious(玫瑰粉)` 暂不用于宠物身体。
   - 后果② 性格养成系统(evolution.md)是后续子系统,本期不做。
5. 旧 JellyMate 残留全部删除。
6. 点击水母 = 开/关面板。
7. Figma:本期不用(无现成稿,且面板已有代码原型);留给后续画新界面。

## A. 动画清理(只保留基础)

保留:基础浮动、拖动、眼睛追光标、点击开关面板、**3 态在场动画(安静/开心/想你,按上次互动时间切换,见决策 4)**。

删除:hover→swimOut、靠近→curious、戳→poked、连戳、专注模式 focus、庆祝转圈、说话气泡 speak、新手教程 tutorial、单/双/三击分支、recall 边缘吸附。

说明:旧的「闲置→sleepy / 夜间→sleepy」其实是同类逻辑,**用文档的 3 态在场机制取代它**(开心=近期互动 / 安静=2h 无互动 / 想你=1天 无互动)。
结果:水母安静浮着,点开面板、再点收起;长时间不理它会从开心→安静→想你。

## B. 架构:面板作为同层浮层

- 把 diary-tab-v5 的面板标记(HTML)+ 样式(CSS,需加作用域防止和水母样式打架)+ 脚本(走势图/tab 切换/日历)抽出来,嵌进 `index.html`,默认隐藏。
- 点水母 → 切换面板显隐;面板朝**空间大的一侧**弹出(interactions-v2 朝向逻辑),始终在视口内。
- 鼠标穿透:面板打开时该区域要能点击,复用已有的 `panel-open` / `panel-close` IPC(`setIgnoreMouseEvents`)。
- 不新建 BrowserWindow,面板天然贴着水母、跟随水母。

## C. 串联后的完整链路(空壳)

```
桌面水母(浮动)─点击─▶ 弹出面板
                         ├─ 聊天 Tab：消息流 + 输入框 + 右侧 ✦(生成今日情绪卡)
                         └─ 日记 Tab：情绪走势图 + 列表/日历 + 周/日/月卡
                       再点水母 ─▶ 收起面板
```

## D. 本期空壳明确不做(标记,避免误以为遗漏)

- 聊天不接真 Claude(打字有回显,回复是预设/脚本)
- ✦ 不真生成卡(走剧本动画,出预设卡)
- 性格养成系统(决策 4 的完整逻辑)

## 交互逻辑核对结论(三方:文档 / interactions-v2 / 当前 index.html)

- **interactions-v2 = 照文档画的,合规**,作为行为真相源(✦生成卡流程、开机三态首句、点击开关面板、冒泡不追问、明信片软送达)。
- **当前 index.html = 上个产品 JellyMate 的交互,基本推倒**:点击语义错(swimOut/专注/连戳)、状态轴错(curious/poked/sleepy/focus vs 文档的安静/开心/想你)、有违反 soul 的卖萌气泡、缺全部情绪小兽核心交互。
- **三个「剧本≠真实」的坑(接的时候避开):**
  1. 剧本冒泡是 `setTimeout(400ms)` 开机自动弹 = demo 演示;真实触发是「日卡生成完成后」才冒泡,别照搬时机。
  2. 剧本里的"面板"是简版浮层 float-panel,**不是**要用的 diary-tab-v5;面板用 diary-tab-v5,只借剧本的弹出方向/冒泡/明信片行为。
  3. diary-tab-v5 聊天 tab 没有 ✦,需把 ✦ 加到其聊天输入框右侧(决策 3)。
