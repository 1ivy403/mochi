narrative.md​
概述​
情绪叙事是情绪小兽区别于普通情绪记录工具的核心能力。它不分析用户，不总结规律，而是把日常
对话中的碎片，在合适的时间维度上，还原成一个让用户认出自己的故事。​
让用户潸然泪下的不是文字好看，而是在故事里突然看见了一直以来的自己。​
写作的根本原则​
让事实说话，不替用户解读。​
洞察是用户读完之后自己产生的，不是被告知的结论。写得越"有洞察感"，越容易变成分析，越让人
感到被研究而不是被陪伴。​
正确的方向：还原细节，然后停下来。​
字数弹性原则：上述字数是上限参考，不是必须填满。生成前先看这个周期的情绪卡片与对话素材是
否撑得起。素材不足时主动缩短，宁可短而真实，不要靠虚泛描写凑字数——凑字数会把叙事拖向空
洞，反而降低质量。​
一、每日情绪卡片​
定位： 今天的一个切面。不求完整，不求深刻，只求真实。​
写作规范：​
•
字数：2-4 句话​
•
视角：第一人称​
•
内容：从今天的对话里找一个具体的细节或场景，还原它​
•
禁止：解读情绪、下结论、说教、升华​
示例对比：​
❌ 今天情绪焦虑，主要因为工作压力。那种累，不只是身体的。​
✅ 今天开了三个小时的会。走出会议室，外面的光有点刺眼。​
判断标准： 读完之后，用户自己感受到了那个状态，而不是被告知那个状态是什么。​

二、每周叙事​
定位： 这一周有没有一个反复出现的情绪或场景。不是卡片的堆叠，而是找到这周的一条线。​
写作规范：​
•
字数：100-200 字​
•
视角：第一人称​
•
内容：从这周的情绪卡片和对话里，找到一个反复出现的情绪、场景或关键词，用叙事的方式说出
来​
•
不需要覆盖这周所有的事，只需要说那个最有重量的​
•
禁止：罗列事件、打分、情绪分析报告​
示例：​
这一周，我好像一直在等什么事情结束。周一的会议、周三的截止日期、周五下班前的那条消息——
每一件事都压着，但没有一件真正过去。直到周六早上睡到自然醒，才想起来，原来我也会有这样的
时刻。​
判断标准： 读完之后，用户说"对，就是这种感觉"，而不是"嗯，是的，这周发生了这些事"。​
三、每月故事​
定位： 情绪叙事最核心的产出。有足够的时间跨度来浮现模式，有足够的素材来具体，有足够的距离
让用户看见自己这个月真正在经历什么。​
写作规范：​
•
字数：400-600 字​
•
视角：第一人称​
•
结构：​
◦开头：这个月反复出现的一个情绪或场景，作为入口​
◦中间：具体的时刻，让模式在叙事里自然浮现，不点破​
◦结尾：一句话点明这个月的主题，但不是结论，是一个让用户自己确认的句子​
•
核心任务：帮用户看见这个月身上反复出现的模式，并在叙事里自然带出它的来龙去脉​
•
禁止：直接说"你这个月的模式是……"、情绪评分、任何分析性语言​
示例：​
十一月就这样过去了。​
三次深夜，两次沉默，一次终于说出口的委屈。每次都是因为同一件事——觉得自己没有被好好看
见。有时候是在会议上，有时候是在一段对话里，有时候只是看着手机发了很久的呆。​

十一月初，我说"有点烦"。十一月中，我说"算了"。十一月底，我没有说什么，只是很早就睡了。​
这种感觉不是第一次了。但这个月，它来得比较密。​
也许十二月，可以试着让自己被看见一次。​
判断标准： 读完之后，用户感到被理解，同时认出了一个一直存在但从未被说出来的自己。​
生成时机​
软送达：不弹窗、不打断；下次打开面板时在聊天里以明信片呈现，日记置顶标未读点。​
与 memory 的关系​
每月故事生成时，AI 同时评估：这个月浮现的模式，是否需要写入 mid_term_memory  或 
long_term_memory 。​
这是情绪叙事反哺记忆系统的唯一入口——故事不只是给用户看的，也是 Mochi 认识用户的方式。​
